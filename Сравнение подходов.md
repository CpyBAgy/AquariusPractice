# Сравнение подходов к тестированию
### Pytest + WebDriver
**Плюсы:**
- Распрострененность подхода
- Много документации и большое сообщество
- Гибкость и интеграция со многими библиотеками
- Проще для понимания

**Минусы:**
- Повторяющийся код
- Сложность поддержки
- Может стать запутанно если тестов много
- Нужно скачивать отдельный WebDriver для кажого браузера

### Pytest + WebDriver + Page Objects
**Плюсы:**
- Переиспользование кода
- Проще поддерживать и масштабировать
- Много документации и большое сообщество
- Лучшая организация кода

**Минусы:**
- Абстракции (с ними могут возникнуть проблемы если не очень хорошо организовано все)
- Архитектура (больше времени на проектирование)
- Может стать слишком зависимо от Page Objects (?)
- Нужно скачивать отдельный WebDriver для кажого браузера

### Playwirght
**Плюсы:**
- Асинхронка/синхронка на выбор
- Есть плагин pytest-playwirght
- Развивается
- Встроено ожидание элементов
- Можно создать несколько контекстов в одном браузере
- Единый API на все браузеры (все будет работать одинаково независимо от браузера)
- Расширенная трассировка (Запись видео и скринов)
- Не требует доп установки драйверов для браузеров (Chromium, Firefox и WebKit под капотом)

**Минусы:**
- Менее стабилен и меньше сообщество, но все активно развивается
- Поддерживает не все браузеры
- Меньше кодовая база
- Может быть более требователен к ресурсам

### RobotFramework + WebDriver
**Плюсы:**
- Встроенная отчетность
- Завязан на ключевые слова

**Минусы:**
- Debug (нужно пользоваться отдельным дебагом, не питонским)
- Завязан на ключевые слова
- Меньшая гибкость

### Pyppeert (Обертка над Puppeteer)
**Плюсы:**
- Асинхронка и только она
- Автоматически скачивает и управляет Chromium

**Минусы:**
- Работает только на Chromium
- Обертка...
- Меньше сообщество и документация

# Критика подхода Pytest + WebDriver + Page Objects
- Могут возникать проблемы с абстракциями
- Определение функциональности Object'ов
- Сложно с поддержкой современных UI (react, vue, SPA)
